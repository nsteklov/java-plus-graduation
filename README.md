Создаваемре приложение — афиша. В этой афише можно предложить какое-либо событие от выставки до похода в кино и собрать компанию для участия в нём.

# Сервисы

## Сервис статистики

Функционал сервиса статистики содержит:
- запись информации о том, что был обработан запрос к эндпоинту API;
- предоставление статистики за выбранные даты по выбранному эндпоинту.

 Спецификация сервиса статистики содержится здесь: [ewm-stats-service.json](https://raw.githubusercontent.com/yandex-praktikum/java-explore-with-me/main/ewm-stats-service-spec.json/ "спецификация сервиса статистики")

## Основной сервис

Содержит всё необходимое для работы продукта.
API основного сервиса разделено на три части:
- **публичная** будет доступна без регистрации любому пользователю сети;
- **закрытая** будет доступна только авторизованным пользователям;
- **административная** — для администраторов сервиса.

Спецификация основного сервиса (за исключением дополнительного сервиса комментариев) содержится здесь: [ewm-main-service-spec.json](https://raw.githubusercontent.com/yandex-praktikum/java-explore-with-me/main/ewm-main-service-spec.json/ "спецификация основного сервиса")

Основной сервис состоит из нескольких сервисов:

### Сервис пользователей

Данный сервис содержит функционал управления пользователями:

- добавление новых пользователей;
- удаление пользователей;
- получение существующих пользователей;
- проверка существования пользователей.

### Сервис событий

Данный сервис является ключевым и содержит основной функционал приложения.
Сервис включает в себя следующий функционал:

- Управление событиями. Добавление новых событий, изменение существующих событий и получение информации о добавленных событиях с различными отборами и с сортировкой по количеству просмотров, которое будет запрашиваться в сервисе статистики, либо по датам событий;
- Управление категориями. Каждое событие должно относиться к какой-либо категории. Имеется возможность добавлять новые категории, а также удалять и изменять существующие категории;
- Подборки событий. Должна быть настроена возможность получения всех имеющихся категорий и подборок событий (такие подборки будут составлять администраторы ресурса).

### Сервис управления заявками на участие

- настроена подача заявок на участие в интересующих мероприятиях;
- доступно изменение статуса (подтверждение, отмена) запроса на участие в событии;
- имеется возможность получения информации о заявках текущего пользователя на участие в чужих событиях;
- имеется возможность получения информации о запросах на участие в событии текущего пользователя.

### Сервис комментариев

* Основное

Система позволяет участникам событий оставлять комментарии к завершенным событиям. 
Все комментарии проходят обязательную модерацию перед публикацией.

- Модель Comment

id - уникальный идентификатор комментария
text - текст комментария (1-1000 символов)
event - событие, к которому оставлен комментарий
author - автор комментария
created - дата и время создания
status - статус комментария (PENDING, PUBLISHED, REJECTED)

- CRUD операции

Создание комментария: 
POST /users/{userId}/comments?eventId={eventId}

Удаление комментария: 
DELETE /users/{userId}/comments/{commentId}

Получение комментариев пользователя: 
GET /users/{userId}/comments?from=0&size=10

Получение опубликованных комментариев события: 
GET /comments/events/{eventId}?from=0&size=10

* Интеграция

- Public API

GET /comments/events/{eventId}

Получение опубликованных комментариев события
Пагинация: from, size
Только комментарии со статусом PUBLISHED

- Private API

POST    /users/{userId}/comments?eventId={eventId}
DELETE  /users/{userId}/comments/{commentId}
GET     /users/{userId}/comments

Создание комментариев к событиям
Удаление своих комментариев
Просмотр своих комментариев

- Admin API

GET    /admin/comments/moderation
PATCH  /admin/comments/{commentId}/moderate

Просмотр комментариев на модерации
Одобрение/отклонение комментариев

* Логика

- Создание комментария
Проверка существования пользователя и события
Проверка, что событие опубликовано
Проверка, что пользователь участвовал в событии (имел подтвержденную заявку)
Создание комментария со статусом PENDING
Сохранение в базу данных

- Модерация комментария
PATCH /admin/comments/{commentId}/moderate?approve=true
approve=true - комментарий публикуется (статус PUBLISHED)
approve=false - комментарий отклоняется (статус REJECTED)

- Удаление комментария
Только автор может удалить свой комментарий
Полное удаление из базы данных

* Дополнительно
- Модерация
Все комментарии создаются со статусом PENDING
Администраторы видят список комментариев на модерации
Администратор принимает решение: опубликовать или отклонить
Опубликованные комментарии видны всем пользователям

- Интеграция с событиями
В EventFullDto добавлено поле commentCount
Автоматический подсчет опубликованных комментариев
Не требуется хранение счетчика в таблице событий

- Ограничения
Максимальная длина комментария: 1000 символов
Минимальная длина: 1 символ
Комментарии только к завершенным событиям
Комментарии только от участников событий
Все комментарии проходят модерацию

# Взаимодействие между сервисами

Для взаимодействия основного сервиса и сервиса статистики используется DiscoveryClient. Вся функциональность и настройки описаны в классе ru.practicum.StatsClient.
Для взаимодействия сервисов внутри основного сервиса используются feign-клиенты. Все методы, необходимые для коммуникации между сервисами, помещены в специально созданных внутренних контроллерах. 

- Сервис пользователей. Класс ru.practicum.UserInternalController.
  
GET  /internal/users/check
GET  /internal/users/{userId}

Проверка существования пользователя с заданным id
Получение пользователя с id

- Сервис событий.  Класс ru.practicum.EventInternalController.

GET  /internal/events/check
GET  /internal/events

Проверка существования события с заданным id
Получение события с id

- Сервис управления заявками на участие.  Класс ru.practicum.RequestInternalController.

GET  /internal/requests/count
GET  /internal/requests/{userId}/{eventId}/check

Получение подтвержденных заявок на участие по событиям
Проверка наличия заявок на участие с заданными ИД пользователя, ИД события и статусом
